Kkom-Morning을 꼬미에게 진짜 도움이 되는 앱으로

아침을 여는 따뜻한 보살핌 — 개선 아이디어 20가지 상세 보고서
작성: 우댕님을 위한 정리 · 2026-05-29 · 커플앱·미세먼지앱·모닝루틴앱 리서치 기반

📌 이 보고서는

Kkom-Morning은 우댕님이 짝꿍 꼬미를 위해 만든 “아침 companion 앱”입니다. 현재 핵심은 오늘의 편지 💌 · 날씨 · 미세먼지 · 옷차림 · 추억 사진 · 둘이 잇는 릴레이 소설 · D-day 카운터예요. 최근 편지는 GAS(느림) → Firestore 실시간(즉시)으로 옮겨 빨라졌습니다.

핵심 인사이트 두 가지를 모든 아이디어의 기준으로 삼았습니다.

① 꼬미는 미세먼지를 가장 중요하게 본다 (미세미세 애용) ② 매일 아침 한 번 여는 앱 — 빠르고, 따뜻하고, 손이 덜 가야 한다

현재 GAS의 날씨·미세먼지 데이터가 null(죽음) 상태라, 아래 여러 아이디어는 “데이터를 살리는 것”에서 출발합니다.

🧭 설계 원칙 (모든 기능의 공통 기준)

미세먼지 우선꼬미의 1순위. 항상 빠르고 정확하게, 화면 맨 위에.
제로 마찰로그인 한 번, 아침에 열면 끝. 입력·설정 최소화.
감정의 따뜻함유틸리티 + 편지·기분 교류. 매일 챙김받는 느낌.
폰 우선(PWA)홈 화면 설치 + 알림 + 위젯. 진짜 앱처럼.
탈(脫)GAS죽은 GAS 데이터를 직접 API/Firestore로 대체해 신뢰성↑.
둘만의 공간두 사람 전용. 사적이고 안전하게(보안 규칙 필수).
A

🌫️ 미세먼지 & 건강 — 꼬미 최우선

5가지
1

실시간 미세먼지 직접 연동 (에어코리아)

죽은 GAS 대신 공공데이터포털에서 직접 받아 1초 미만으로 PM10/PM2.5·등급 표시.
난이도 보통우선순위 ★★★★★참고: 미세미세 · AirKorea
왜 좋나
꼬미가 매일 제일 먼저 보는 정보인데 지금은 “정보 없음”. 이걸 살리는 게 앱 부활의 핵심.
구현
Next.js /api/air 라우트 → 에어코리아 “측정소별 실시간 측정정보” 호출(호평동/남양주 측정소). data.go.kr 무료 키 1개 필요. 응답을 60초 캐시.
2

시간별 · 내일 미세먼지 예보

“오후부터 나빠져요”처럼 24시간·이틀 예보를 보여줘 하루 계획에 도움.
난이도 보통우선순위 ★★★★☆미세미세도 24h/2일 예보 제공
왜 좋나
“지금”뿐 아니라 “언제 나가면 좋은지”를 알려줘 외출·환기 타이밍을 잡게 해줌.
구현
에어코리아 “대기질 예보통보” + 시간별 측정 추세. 미니 막대 그래프(이미 recharts 설치됨)로 시각화.
3

미세먼지 푸시 알림 🎯

‘나쁨’ 이상이면 아침에 “오늘 미세먼지 나쁨, 마스크 챙겨요 😷”. 좋아지면 “오늘 공기 깨끗! 산책 추천 🌿”.
난이도 높음우선순위 ★★★★★PWA 설치 전제
왜 좋나
앱을 열지 않아도 꼬미가 챙김받는 핵심 기능. “미세먼지 중요”에 가장 직접적으로 응답.
구현
PWA(아이디어 11) + 웹푸시(VAPID/web-push) 또는 FCM. Vercel Cron이 매일 아침 등급 체크 → 임계값 넘으면 발송. ⚠️ iOS는 16.4+ & 홈 화면 설치 시에만 가능.
4

마스크 & 환기 추천

등급에 따라 “KF94 권장” / 공기 좋을 때 “지금 환기하기 좋아요 🪟”.
난이도 쉬움우선순위 ★★★☆☆
왜 좋나
숫자를 “행동”으로 번역. 미세미세의 ‘활동 추천’과 같은 결.
구현
미세먼지 카드 안에 등급→문구 매핑만 추가(데이터 1·2번에 의존). 코드 거의 없음.
5

계절 건강 지표 (꽃가루·자외선·체감온도)

봄엔 꽃가루, 여름엔 자외선, 겨울엔 체감온도까지 한 카드에.
난이도 보통우선순위 ★★★☆☆
왜 좋나
“건강 챙김” 테마 확장. 꼬미가 알레르기·피부에 민감하면 특히 유용.
구현
기상청 생활기상지수 API(자외선·꽃가루·체감온도, 모두 data.go.kr). 미세먼지 키와 같은 포털.
B

💌 편지 & 마음 교류 — 둘의 핵심

5가지
6

편지 도착 푸시 알림

우댕이 편지를 보내면 꼬미 폰에 즉시 “💌 오늘의 편지가 도착했어요”.
난이도 높음우선순위 ★★★★★
왜 좋나
지금은 앱을 열어야만 편지를 봄. 알림이 있으면 “설렘”이 실시간이 됨 — 앱의 감정적 심장.
구현
Firestore onCreate(Cloud Function) 또는 편지 전송 시 서버에서 상대에게 웹푸시. 알림 인프라(11·3과 공유).
7

편지 보관함 · 타임라인

지금은 최신 1통만. 그동안 주고받은 편지를 날짜별로 모아 다시 읽기.
난이도 쉬움우선순위 ★★★★☆참고: Between 메모리 타임라인
왜 좋나
쌓인 편지는 그 자체로 추억. 힘든 날 다시 꺼내 보는 보물상자가 됨.
구현
이미 Firestore letters에 다 저장 중. /letters 페이지에서 orderBy(createdAt)로 목록만 그리면 됨. 데이터 이미 있음 → 화면만.
8

음성 편지

글 대신 목소리로. 출근길에 듣는 우댕의 한마디.
난이도 보통우선순위 ★★★☆☆
왜 좋나
텍스트보다 따뜻함. 목소리는 감정 전달이 압도적.
구현
녹음(MediaRecorder) → Firebase Storage 업로드 → 편지 문서에 오디오 URL. (앱에 이미 있는 TTS flow로 ‘텍스트→음성’ 변환도 가능.)
9

오늘의 기분 공유

이모지/한 줄로 컨디션 교환. 꼬미가 안 좋으면 우댕이 바로 알아챔.
난이도 쉬움우선순위 ★★★★☆참고: LuvDiary mood
왜 좋나
“오늘 어때?”를 매일 자연스럽게. 멀리 있어도 서로의 상태를 안다.
구현
Firestore moods 컬렉션(편지와 동일 패턴). 홈 상단에 둘의 오늘 기분 이모지 2개.
10

예약 편지 · 타임캡슐

미래 날짜에 ‘도착’하는 편지. 기념일·생일에 깜짝 등장.
난이도 보통우선순위 ★★★☆☆
왜 좋나
“미래의 나/너에게”라는 설렘. 1주년 같은 날 자동으로 열림.
구현
편지에 openAt 필드 → 그 시각 전엔 잠금 표시, 이후 노출. (정렬/필터만 추가.)
C

🔔 알림 · 습관 · 접근성 (PWA)

4가지
11

PWA — 홈 화면에 추가

진짜 앱처럼 폰에 설치. 전체화면 실행 + 오프라인에서도 최근 편지 캐시.
난이도 보통우선순위 ★★★★★ (기반)
왜 좋나
모든 알림(3·6·13)의 전제조건. 매일 여는 앱이라면 홈 화면 아이콘이 필수. 설치만으로 “우리 앱” 느낌.
구현
manifest.json(아이콘·standalone) + 서비스워커. Next.js PWA 가이드대로. Vercel은 HTTPS라 조건 충족.
12

홈 화면 / 잠금화면 위젯

앱을 안 열어도 미세먼지 등급 + ‘편지 도착’ 여부를 위젯으로.
난이도 높음우선순위 ★★★☆☆참고: Cupla · Lovio 위젯
왜 좋나
“열어야 보는” 마찰 제거. 미세먼지를 폰 첫 화면에서 바로 확인.
구현
웹은 한계 → 진짜 위젯은 네이티브 영역. 대안: iOS 스크린샷 위젯/단축어 또는 추후 경량 네이티브 래퍼. (중장기)
13

모닝 브리핑 알림

매일 아침 정해진 시간 한 번에: “좋은 아침 꼬미 💚 미세먼지 보통, 편지 도착, 13°C”.
난이도 높음우선순위 ★★★★☆
왜 좋나
앱의 콘셉트(아침 companion)를 한 줄로 완성. 하루를 챙김으로 시작.
구현
Vercel Cron(매일 07:00) → 미세먼지+편지+날씨 묶어 웹푸시. 알림 인프라 공유(3·6).
14

접속 스트릭 · 함께 출석

둘이 매일 앱을 연 연속 일수. 귀여운 보상/뱃지로 습관화.
난이도 쉬움우선순위 ★★☆☆☆참고: TwoDo 마일스톤 스트릭
왜 좋나
“우리 며칠째 함께”라는 작은 게임. 재방문 동기 + 소소한 재미.
구현
Firestore에 마지막 접속일·연속일수 기록. 홈에 작은 🔥 카운터.
D

🌅 아침 일상 도움

4가지
15

옷차림 추천 고도화 + 날씨 직접 연동

실제 기온·미세먼지·강수를 반영: “쌀쌀+미세먼지 나쁨 → 마스크 + 가디건”.
난이도 보통우선순위 ★★★★☆
왜 좋나
아침에 제일 실용적인 “뭐 입지?”를 해결. 죽은 날씨 데이터도 함께 부활.
구현
기상청 단기예보 API(data.go.kr)로 날씨 직접 호출 → 기온·강수+미세먼지 등급을 규칙으로 추천. (지금 옷차림은 GAS·AI flow였지만 둘 다 사실상 미작동.)
16

물 · 약 · 영양제 리마인더

“물 한 잔 💧” “영양제 챙겼어?” 같은 부드러운 챙김 알림.
난이도 보통우선순위 ★★★☆☆참고: Me+ 셀프케어 루틴
왜 좋나
건강 챙김의 디테일. 우댕이 직접 못 챙겨도 앱이 대신 챙겨줌.
구현
간단한 리마인더 설정 + 웹푸시. 또는 모닝 브리핑(13)에 끼워넣기.
17

생리주기 케어 (선택 · 꼬미 동의 하에)

우댕이 주기를 알고 미리 챙김: “곧 그날, 따뜻한 거·진통제 챙겨주기”.
난이도 보통우선순위 ★★★★☆참고: Flo for Partners · Blood for Couples
왜 좋나
파트너 케어 앱의 핵심 기능. “챙김받는다”는 느낌이 가장 큰 영역. 매우 사적이니 꼬미가 원할 때만.
구현
꼬미가 시작일 기록 → 예측 + D-day. 우댕 화면엔 “곧/오늘” 부드러운 안내. Firestore 비공개 컬렉션 + 보안 규칙 엄격히.
18

둘이 보는 캘린더 · 할 일

데이트·기념일·약속·할 일을 한 곳에서 공유.
난이도 보통우선순위 ★★★☆☆참고: Between · Cupla 캘린더
왜 좋나
커플앱 단골 기능. 약속을 놓치지 않고, 다가올 일정으로 설렘.
구현
Firestore events 컬렉션 + 월간 뷰(react-day-picker 이미 설치). 기념일은 D-day(19)와 연동.
E

🎁 추억 · 기념일 · 재미

2가지
19

D-day 확장 + 기념일 알림 + 추억 갤러리 살리기

100일·1주년 자동 카운트 + 미리 알림. 잠자는 추억 사진 갤러리 부활.
난이도 쉬움우선순위 ★★★☆☆
왜 좋나
이미 ‘2023-09-28 since’ 카운터가 있음. 100일·기념일 알림은 커플앱 표준(Lovewick은 1주 전 알림).
구현
여러 기념일 등록 + 임박 알림(브리핑에 포함). MemoryGallery 컴포넌트는 이미 있으니 사진 소스만 Firebase Storage로 연결.
20

데일리 퀴즈 / 대화 카드 살리기

이미 만들어둔 퀴즈 UI를 화면에 연결: “서로 알아맞히기” + 대화 주제 카드.
난이도 쉬움우선순위 ★★★☆☆참고: Lovewick 디스커버리 카드
왜 좋나
kkom-quiz·QuizCard·/api/quiz가 멀쩡히 있는데 어느 화면에도 안 붙어있음. 연결만 하면 바로 기능 부활.
구현
홈에 퀴즈 카드 렌더 추가(이미 만든 컴포넌트 import). 데이터 소스만 Firestore로 옮기면 더 안정적.

🗺️ 추천 실행 순서 (로드맵)

효과 대비 난이도로 정렬했습니다. 꼬미의 1순위(미세먼지)와 이미 80% 만들어진 것(편지 보관함·퀴즈)부터 빠르게 가는 걸 권합니다.

단계무엇을왜 먼저
1 · 빠른 승리#1 미세먼지 실연동, #7 편지 보관함, #20 퀴즈 살리기, #4 마스크·환기 문구, #9 기분 공유임팩트 크고 난이도 낮음. 미세먼지는 꼬미 1순위, 보관함·퀴즈는 거의 다 만들어져 화면만 붙이면 됨.
2 · 기반 + 알림#11 PWA → #3 미세먼지 알림 · #6 편지 알림 · #13 모닝 브리핑, #15 날씨·옷차림, #2 미세먼지 예보PWA가 알림의 전제. 한 번 깔면 ‘앱을 안 열어도 챙겨주는’ 단계로 도약.
3 · 확장#17 생리주기 케어, #18 캘린더, #8 음성편지, #10 예약편지, #5 계절지표, #16 리마인더, #19 기념일, #14 스트릭, #12 위젯깊이를 더하는 기능들. 1·2단계가 자리 잡은 뒤 하나씩.
